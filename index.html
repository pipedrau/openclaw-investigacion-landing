Estrategia técnica y financiera
para operar con USD 40–50/mes

El costo se dispara por contexto inflado + respuestas largas + sub-agents sin límites. La solución real no es “cambiar de modelo”, sino imponer gobernanza de tokens y presupuesto por diseño.

Objetivo mensual
USD 40–50
Uso de referencia
8 h/día
Palanca crítica
Hard cap

Riesgo de sobrecosto

Arquitectura recomendada

  • Manager (Telegram)
    Triaging, contexto corto, herramientas mínimas.
  • Specialist sub-agents
    Dev/deploy/research en sesiones aisladas, con aprobación.
  • Node Mac mini
    Ejecución local de CLIs/repos, separando cerebro de acción.
  • GitHub Actions
    workflow_dispatch para pipeline y logs.
  • Netlify/Vercel
    Build hooks/API sin depender de navegación UI.
  • Governance
    Allowlists, approvals, límites por rol.

Recomendación ejecutiva

  • LiteLLM + virtual keys con presupuesto mensual duro por rol.
  • Routing por agente: manager barato, especialistas ultra baratos, premium solo por excepción.
  • Reducción de tokens: skills mínimas, prompts compactos, respuestas cortas por defecto.

Plan de implementación

Fase 1 (esta semana): topes de gasto, políticas de salida/contexto, dashboard diario de tokens/costo.

Fase 2 (próxima): node Mac mini, catálogo de sub-agents, playbooks deploy/diagnóstico por API.

Entrega esperada: operación estable con control de costo y trazabilidad por agente.
